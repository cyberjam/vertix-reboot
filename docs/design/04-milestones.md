# 개발 마일스톤 (Development Milestones)

> 기준 문서: [01-game-analysis](./01-game-analysis.md) · [02-mvp-scope](./02-mvp-scope.md) · [03-technical-design](./03-technical-design.md)
> 스택: Next.js · TypeScript · Phaser 3 · Colyseus · Supabase

## 원칙

- 각 마일스톤은 **1~3시간 내 완료** 가능한 크기.
- 각 마일스톤은 **독립적으로 테스트 가능** (완료 기준에 검증 방법 명시).
- **순차 개발** — 앞 마일스톤이 뒷 마일스톤의 토대가 된다.
- 워크플로우: 마일스톤마다 **브랜치 생성 → 작업 → PR → 자동 머지**.

범위는 MVP(클래스 3종 / 히트스캔 무기 / FFA / 1맵 / 리스폰 / 힐팩 / 점수 / 멀티플레이 / HUD)에 한정한다.

---

## Milestone 1 — 모노레포 스캐폴딩
**목표**
pnpm workspaces + Turborepo 모노레포 골격 구성. `apps/web`(Next.js+TS), `apps/game-server`(Colyseus+TS), `packages/shared`(공유 타입) 생성 및 상호 import 연결.

**예상 작업시간**
2시간

**완료 기준**
- `pnpm install && pnpm build` 가 전체 워크스페이스에서 성공.
- `apps/web` 개발 서버가 빈 페이지를 렌더링(`/`).
- `apps/game-server`가 Colyseus 인스턴스로 부팅(포트 리슨 로그 확인).
- `packages/shared`의 더미 export를 web·game-server 양쪽에서 import해 빌드 통과.

---

## Milestone 2 — 공유 데이터/상수 정의
**목표**
`packages/shared`에 `ClassDef`/`WeaponDef`/`MapDef` 타입과 MVP 데이터(클래스 3종: triggerman·hunter·vince, 해당 무기들), 그리고 `constants`(TICKRATE/PATCHRATE 등)를 데이터로 정의.

**예상 작업시간**
1.5시간

**완료 기준**
- 클래스 3종 + 무기 데이터가 타입에 맞게 정의되고 빌드 통과.
- 단위 테스트: 모든 클래스의 `loadout` 무기 ID가 무기 데이터에 실제 존재함을 검증(참조 무결성).
- 데이터를 코드에서 ID로 조회하는 헬퍼(`getClass(id)`, `getWeapon(id)`)가 테스트로 동작 확인.

---

## Milestone 3 — Phaser 클라이언트 셸 + 맵 렌더 (오프라인)
**목표**
Next.js `/play`에 Phaser 3를 SSR 없이 마운트. Arena 씬에서 `MapDef` 타일맵 렌더, 카메라, 로컬 더미 플레이어 스프라이트 표시. 네트워크 없음.

**예상 작업시간**
3시간

**완료 기준**
- `/play` 진입 시 Phaser 캔버스에 맵(벽/통로)이 그려짐.
- 더미 플레이어 스프라이트가 화면에 표시되고 카메라가 따라감.
- SSR 오류 없이(클라 전용 동적 import) 빌드/실행 성공.

---

## Milestone 4 — 로컬 이동 + 클라이언트 충돌
**목표**
WASD 입력 캡처 → 로컬 플레이어 이동, 마우스로 조준각 표현, 맵 벽과의 충돌 처리(클라 측 시뮬). `sim-core`의 이동/충돌 함수 초안 작성.

**예상 작업시간**
2시간

**완료 기준**
- WASD로 캐릭터가 이동하고 마우스 방향으로 조준 방향이 회전.
- 벽을 통과하지 못함(통로 안에서만 이동).
- 이동/충돌 로직이 `packages/shared/sim-core`에 위치해 추후 서버가 재사용 가능한 형태.

---

## Milestone 5 — Colyseus 접속 + 스폰 + 타 플레이어 표시
**목표**
`GameState`/`Player` Schema 정의(shared), `ArenaRoom` 기본 구현(onAuth 게스트 허용·onJoin 스폰). 클라 colyseus.js로 접속하여 자신/타 플레이어를 스프라이트로 표시.

**예상 작업시간**
3시간

**완료 기준**
- 브라우저 탭 2개로 접속 시 서로의 플레이어가 화면에 나타남.
- 한 탭이 나가면(onLeave) 다른 탭에서 해당 플레이어가 사라짐.
- (이동은 아직 동기화 전 — 스폰 위치 표시까지만 확인) 스폰 위치가 맵 스폰 포인트 중 하나.

---

## Milestone 6 — 권위 이동 동기화 + 보간
**목표**
클라가 입력 커맨드(`InputCommand`, seq 포함)를 서버로 전송 → 서버 권위 이동/충돌(`sim-core` 재사용) → 상태 패치 브로드캐스트. 타 플레이어는 스냅샷 보간으로 부드럽게 렌더.

**예상 작업시간**
3시간

**완료 기준**
- 탭 A에서 이동하면 탭 B에서 해당 플레이어가 (보간되어) 부드럽게 움직임.
- 서버 로그로 입력 수신·시뮬·패치 송신이 확인됨.
- 클라가 임의로 위치를 보내도 서버 권위 위치로 덮어써짐(클라 위치 위조 무효).

---

## Milestone 7 — 클라이언트 예측 + 보정(Reconciliation)
**목표**
내 캐릭터는 입력 즉시 로컬 적용(예측), 서버의 `lastProcessedInputSeq` 기준으로 미반영 입력 재적용(보정). 인공 지연 주입 옵션으로 검증.

**예상 작업시간**
2.5시간

**완료 기준**
- 인공 지연(예: 150ms) 주입 상태에서도 내 캐릭터 이동이 즉각적이고 끊기지 않음.
- 서버 보정 시 위치 튐(rubber-banding)이 눈에 띄지 않게 수렴.
- 예측 비활성 토글과 비교해 체감 차이 확인 가능.

---

## Milestone 8 — 사격: 히트스캔 + 데미지 (머신건)
**목표**
`fire` 입력 → 서버 권위 히트스캔 레이캐스트(맵 충돌 + 적 히트박스) → 데미지 적용 → `Player.hp` 동기화. Triggerman 머신건 1종. 피격/발사 1회성 이벤트 메시지로 이펙트 트리거.

**예상 작업시간**
3시간

**완료 기준**
- 한 플레이어가 다른 플레이어를 조준·사격하면 대상 `hp`가 데미지(25)만큼 감소(양 클라에 동기화).
- 벽 뒤의 적은 맞지 않음(레이가 벽에서 차단).
- 발사/피격 시 클라에 시각 이펙트(머즐/히트) 표시.

---

## Milestone 9 — 무기 메커니즘: 탄창/재장전/연사 + 무기 3종
**목표**
`WeaponState`(ammoInMag/reloading/nextFireAt) 구현. `fireRateMs`·`magSize`·`reloadMs` 적용, 예비탄 무한. 스나이퍼(단발), 샷건(다중 펠릿 히트스캔) 추가, 무기 전환(주/보조).

**예상 작업시간**
3시간

**완료 기준**
- 연사 속도 제한·탄창 소진·자동/수동 재장전이 서버 권위로 동작.
- 스나이퍼 단발 고데미지, 샷건 다중 펠릿 산탄이 각각 의도대로 명중 판정.
- Hunter 보조무기(머신피스톨) 전환이 동작.

---

## Milestone 10 — 사망 / 리스폰 / 클래스 선택
**목표**
HP 0 → 사망 처리(킬 귀속). 리스폰 딜레이 + 안전 스폰 선택 + 스폰 무적. 리스폰 화면에서 클래스 변경 → HP/무기 재설정.

**예상 작업시간**
2.5시간

**완료 기준**
- 피격으로 HP 0이 되면 사망 상태가 되고, 설정된 딜레이 후 안전한 스폰 지점에서 부활.
- 스폰 직후 짧은 무적이 적용되어 즉시 스폰킬되지 않음.
- 리스폰 시 다른 클래스를 선택하면 해당 클래스의 HP/무기로 부활(서버 검증).

---

## Milestone 11 — 힐팩 시스템
**목표**
`Pickup` Schema. 맵 고정 위치 힐팩, 접촉 시 즉시 회복(+50, 최대치 한도), 픽업 후 쿨다운 뒤 재스폰.

**예상 작업시간**
1.5시간

**완료 기준**
- 손상된 플레이어가 힐팩에 닿으면 HP가 회복되고 힐팩이 사라짐(양 클라 동기화).
- 쿨다운 경과 후 같은 위치에 힐팩이 재등장.
- 풀피 플레이어는 회복 효과 없음(또는 픽업 안 함) — 규칙대로 동작.

---

## Milestone 12 — FFA 모드 + 점수 + 라운드 루프
**목표**
모드 규칙 플러그인 구조로 FFA 구현: 킬 100점, 목표 1500점 또는 시간(4분) 종료. `MatchState`(phase/timeRemaining/score). 라운드 종료 → 결과 → 다음 라운드 재시작.

**예상 작업시간**
3시간

**완료 기준**
- 킬 시 점수가 100점씩 누적되고 동기화됨.
- 누군가 1500점 도달 또는 타이머 종료 시 라운드가 종료되고 승자가 결정됨.
- 라운드 종료 후 상태가 초기화되어 새 라운드가 시작됨.

---

## Milestone 13 — HUD
**목표**
Phaser/React HUD: 체력바, 현재 탄창/재장전 표시, 점수판(킬 리더보드), 라운드 타이머, 킬 피드.

**예상 작업시간**
2.5시간

**완료 기준**
- 체력·탄창·점수·남은 시간이 실시간 상태와 일치하게 표시됨.
- 킬 발생 시 킬 피드에 항목이 추가됨.
- 리더보드가 점수 순으로 정렬되어 갱신됨.

---

## Milestone 14 — 랙 보상(Lag Compensation)
**목표**
서버에 최근 N틱 위치 스냅샷 링버퍼 보관. 히트스캔 판정 시 사수 RTT/2만큼 표적 위치 되감아 명중 판정.

**예상 작업시간**
3시간

**완료 기준**
- 인공 지연 주입 상태에서 "조준한 대로" 명중률이 보상 미적용 대비 개선됨(비교 측정).
- 되감기 윈도 상한이 설정되어 과도한 보상(부정확) 방지.
- 보상 on/off 토글로 차이를 검증 가능.

---

## Milestone 15 — Supabase 인증(게스트/로그인) + 닉네임
**목표**
Supabase Auth 연동(익명 세션 + 이메일/OAuth 택1). 클라 access_token을 `onAuth`에서 검증. 닉네임 입력·표시. 비로그인 게스트 허용.

**예상 작업시간**
2.5시간

**완료 기준**
- 게스트(닉네임만)와 로그인 유저 모두 입장 가능.
- 로그인 유저의 JWT가 서버에서 검증되어 신뢰 가능한 식별자 확보(위조 토큰 거부).
- 플레이어 닉네임이 다른 클라에 표시됨.

---

## Milestone 16 — 경기 결과 기록 + 배포
**목표**
Supabase `matches`/`match_participants` 마이그레이션 + RLS. 라운드 종료 시 게임 서버(service_role)가 결과 기록(로그인 유저). 배포(web=Vercel, game-server=상태유지 호스트).

**예상 작업시간**
3시간

**완료 기준**
- 라운드 종료 후 `matches`/`match_participants`에 결과 행이 생성됨(로그인 유저).
- RLS로 클라이언트 직접 쓰기는 거부되고, 서버 기록만 허용됨.
- 배포 환경의 공개 URL로 접속해 4분 FFA를 끝까지 플레이 가능(MVP Definition of Done 충족).

## 마일스톤 의존 순서 요약

```
M1 스캐폴딩
└ M2 공유 데이터
  └ M3 Phaser 맵 렌더 ─ M4 로컬 이동/충돌
       └ M5 Colyseus 접속/스폰
            └ M6 권위 이동 동기화 ─ M7 예측/보정
                 └ M8 히트스캔 사격 ─ M9 무기 메커니즘
                      └ M10 사망/리스폰 ─ M11 힐팩
                           └ M12 FFA/점수/라운드 ─ M13 HUD
                                └ M14 랙 보상
                                     └ M15 인증 ─ M16 결과기록/배포
```

> M1~M13까지가 "플레이 가능한 MVP 코어", M14는 손맛 보강, M15~M16은 계정/영속/배포.
> 각 마일스톤은 이전까지의 결과 위에서 독립 검증 가능하도록 완료 기준을 정의했다.
