# Vertix Reboot

[English](./README.md) · **한국어**

몇 초면 바로 뛰어드는 브라우저 멀티플레이어 아레나 슈터. 이름 정하고 클래스만
고르면 곧장 매치에 들어갑니다 — 설치도, 계정도 필요 없습니다.

Vertix Reboot은 [Vertix Online](https://en.wikipedia.org/wiki/Vertix.io)이
보여줬던 그 재미 — 짧은 라운드, 즉시 리스폰, 단순한 조작, 링크 하나로 모이는
친구들 — 을 다시 만들어 보려는 오픈소스 프로젝트입니다. 처음부터 새로 짠
코드베이스이며 지금도 활발히 개발 중입니다.

## 소개

브라우저에서 돌아가는 프리포올(자유 대전) 탑다운 슈터입니다. 다른 플레이어들과
함께 아레나에 스폰돼 싸우고, 죽으면 몇 초 뒤 다시 살아나고, 누군가 점수 한도에
도달하거나 라운드 타이머가 끝날 때까지 계속합니다. 그러면 초기화되고 다시
시작합니다.

**왜 만드나요?** "링크 열면 바로 매치에 들어가는" 브라우저 슈터는 좋은 형식인데
요즘은 잘 안 만들어집니다. 그 감각을, 작업하기 즐겁고 기여하기도 쉬운 스택 위에서
다시 만들어 보려는 시도입니다 — 서버 권위 기반 넷코드, 타입이 잡힌 공유 코어,
깔끔한 React/Phaser 클라이언트.

**목표는** 친구 몇 명과 10분 동안 정말 재밌게 할 수 있는, 작고 솔직한 게임입니다.
빨리 들어가고, 빨리 익히고, 빨리 다시 하는 것. 플랫폼도, 상점도 아닌 — 그저 그
반복되는 재미만.

## 구현된 기능

아래는 모두 구현되어 있으며, 로컬에서 실행하면 바로 플레이할 수 있습니다.

- **즉시 플레이.** `/play`를 열고, 이름 입력하고, 클래스 고르고, **ENTER GAME**.
  로그인도 다운로드도 없습니다. 이름과 클래스는 다음에 다시 올 때 기억됩니다.
- **클래스 3종, 무기 4종.** 각자 플레이 방식이 다릅니다:
  - **Triggerman** — 100 HP, 머신건. 올라운더.
  - **Hunter** — 50 HP, 스나이퍼 + 머신피스톨. 약하지만 원거리에서 치명적.
  - **Vince** — 100 HP, 4펠릿 샷건. 근거리 폭딜.
- **빠른 리스폰.** 죽으면 원하는 경우 다른 클래스로 바꾸고, 약 2초 뒤 다시
  전장으로 돌아옵니다.
- **서버 권위 넷코드.** 이동·사격·피격 판정은 모두 서버가 결정합니다. 다만 내
  캐릭터 이동은 클라이언트에서 예측·보정하므로, 끊기거나 튀지 않고 즉각
  반응합니다.
- **엄폐와 시야 차단.** 벽은 총알과 시야를 모두 막습니다. 벽 너머로 쏠 수도,
  맞을 수도 없습니다.
- **헬스팩** 이 맵 곳곳에 있으며, 사용 후 쿨다운을 거쳐 재생성됩니다.
- **프리포올 라운드.** 킬당 100점. 먼저 1500점에 도달하거나 — 4분 타이머가 끝났을
  때 최고 점수인 — 플레이어가 승리합니다. 그 후 라운드가 초기화됩니다.
- **풀 HUD.** 체력, 탄약, 라운드 타이머, 실시간 리더보드, 킬 피드.
- **미니맵** — 벽, 플레이어, 활성 헬스팩 표시.
- **스코어보드** — **Shift** 를 누르고 있으면 전체 표시. 그리고 사망 오버레이는
  나를 죽인 사람을 보여주고, 클래스를 바꾸게 해주며, 리스폰까지 카운트다운합니다.
- **점프** (**Space**) — 짧은 수직 도약.
- **서버 브라우저.** 가장 적합한 방으로 퀵매치하거나, 열려 있는 방들을 둘러보고
  인원 수를 보며 골라 입장합니다.
- **설정.** 카메라 흔들림과 시각 효과를 끄고 켜고, 조작법을 확인합니다. 설정은
  브라우저에 저장됩니다.

### 조작법

| 키 | 동작 |
|-----|--------|
| `W` `A` `S` `D` | 이동 |
| 마우스 | 조준 |
| 좌클릭 | 발사 |
| `R` | 재장전 |
| `Q` | 무기 전환 |
| `1` / `2` / `3` | 클래스 선택 (리스폰 시 적용) |
| `Space` | 점프 |
| `Shift` | 전체 스코어보드 보기 |
| `Esc` | 오버레이 닫기 |

## 로드맵

### 진행 중

- **공개 호스팅 빌드** — 직접 아무것도 실행하지 않고 브라우저에서 바로 플레이할 수
  있도록. 웹은 Vercel, 게임 서버는 Render에 올리며, 프로덕션용 Docker 이미지·헬스
  체크·CORS를 함께 준비 중입니다.

### 예정

- 사운드 효과
- 추가 클래스 / 무기
- 추가 모드 (예: 팀 데스매치)
- 키 리매핑
- 킬스트릭 표시 ("Double Kill" 등)
- 선택적 계정 / 영속 통계 (도입 여부 검토 중)

구현된 기능은 [구현된 기능](#구현된-기능)에 있습니다. 거기에 없는 것은 아직 만들지
않은 것입니다.

## 기술 스택

| 영역 | 기술 |
|-------|------|
| **프론트엔드** | Next.js 15 (App Router), React 19, TypeScript — UI·메뉴·오버레이. 인게임 렌더링과 입력은 [Phaser 3](https://phaser.io). |
| **백엔드** | [Colyseus](https://colyseus.io) 0.16 권위 게임 서버 (Node 20, TypeScript). |
| **실시간** | Colyseus 스키마 기반 WebSocket 상태 동기화 (시뮬레이션 30Hz, 패치 50ms). 클라이언트는 `colyseus.js` + 이동 예측·보정. |
| **공유 코어** | `@vertix/shared` — 클래스/무기/맵 데이터와 결정론적 이동 시뮬레이션. 클라이언트와 서버가 같은 코드를 단일 진실 공급원으로 사용. |
| **툴링** | pnpm workspaces + Turborepo 모노레포. |
| **배포** | Vercel (웹) + Render (게임 서버) — _예정, 로드맵 참고._ |

## 로컬 개발

**사전 준비:** [Node.js](https://nodejs.org) ≥ 20, 그리고 (`corepack`을 통한)
[pnpm](https://pnpm.io).

```bash
# 1. pnpm이 없다면 활성화
corepack enable

# 2. 의존성 설치
pnpm install

# 3. 웹 클라이언트와 게임 서버를 함께 실행
pnpm dev
```

- 웹 클라이언트 → http://localhost:3000 (랜딩 페이지에 `/play` 링크)
- 게임 서버 → `ws://localhost:2567`

따로 실행하고 싶다면:

```bash
pnpm dev:web      # Next.js 클라이언트만  (:3000)
pnpm dev:server   # Colyseus 서버만        (:2567)
```

**멀티플레이 확인:** `/play`를 브라우저 탭 두 개(또는 브라우저 두 개)에서 열면 같은
아레나 안에 두 플레이어가 보입니다.

**검사:**

```bash
pnpm typecheck    # 모든 패키지 타입체크
pnpm build        # 전체 빌드
```

설계 문서는 [`docs/design`](./docs/design/README.md)에 있습니다.

## 배포

호스팅 플레이는 아직 라이브가 아니며, 현재 로드맵의 최우선 작업입니다. 구상하는
구성은 웹 클라이언트를 **Vercel** (루트 `apps/web`)에, 권위 게임 서버를 **Render**에
Docker 서비스로 올리고, 클라이언트는 `NEXT_PUBLIC_GAME_SERVER_URL`로 서버를
가리키게 하는 것입니다 (HTTPS 페이지에서는 `wss://` 사용). 프로덕션 설정(Dockerfile,
헬스 체크, CORS)도 함께 준비되고 있습니다.

## 기여

작은 인디 프로젝트이며 기여를 환영합니다 — 버그 수정, 게임플레이 조정, 새 무기나
클래스, 또는 그냥 플레이해 보고 어색했던 점을 알려주는 것까지.

1. 저장소를 포크하고 `dev`에서 브랜치를 만듭니다 (예: `feat/your-thing`).
2. 변경을 작업합니다. 게임플레이는 **서버 권위**를 유지하세요 — 클라이언트는 의도를
   보내고, 결과는 서버가 결정합니다.
3. PR을 열기 전에 `pnpm typecheck`와 `pnpm build`를 실행하세요.
4. `dev`를 대상으로 풀 리퀘스트를 엽니다.

클래스나 무기를 추가한다면 `@vertix/shared`에 정의하세요. 그래야 클라이언트와
서버가 한 곳에서 동기화됩니다.
