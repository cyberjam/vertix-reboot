## 이 게임이 재밌는 이유

- ⚡ **바로 입장** — 페이지 열고, 이름 치면 끝. 설치도 가입도 없습니다.
- 🔁 **빠른 라운드** — 죽어도 몇 초 만에 복귀. 먼저 1500점을 찍거나, 4분이 다 됐을 때 최고 점수면 승리. 그리고 다시 시작.
- 🎯 **클래스 전투** — 클래스 3종, 각자 무기와 장단점이 다릅니다.
- 🌐 **진짜 멀티플레이** — 모두가 하나의 아레나에서, 브라우저에서 바로.

---

## 기능

- **클래스 3종, 무기 4종** — 체력을 내주는 대신 화력이나 사거리를 얻습니다:
  - **Triggerman** — 머신건. 무난한 올라운더.
  - **Hunter** — 스나이퍼 + 머신피스톨. 약하지만 거리에서 강력.
  - **Vince** — 4펠릿 샷건. 근접 폭딜.
- **죽으면 클래스 교체** — 리스폰 화면에서 상대를 카운터할 클래스를 고릅니다.
- **엄폐가 중요** — 벽은 총알도 시야도 막습니다.
- **헬스팩** 이 맵 곳곳에 등장합니다.
- **실시간 HUD** — 체력, 탄약, 라운드 타이머, 리더보드, 킬 피드, 그리고 누구한테
  당했는지 보여주는 사망 화면.
- **미니맵** 과 전체 스코어보드(**Shift** 누르고 있기).
- **서버 브라우저** — 가장 적합한 방으로 퀵매치하거나, 둘러보고 인원수 보며 입장.
- **점프**(**Space**), 카메라 흔들림·효과 설정(브라우저에 저장).

## 로컬 개발

[Node.js](https://nodejs.org) ≥ 20, [pnpm](https://pnpm.io) 필요.

```bash
corepack enable      # pnpm이 없다면 활성화
pnpm install
pnpm dev             # 웹 클라이언트 + 게임 서버 동시 실행
```

- 웹 → http://localhost:3000 (랜딩 페이지에 `/play` 링크)
- 서버 → `ws://localhost:2567`

멀티플레이는 `/play`를 탭 두 개로 열어 확인하세요. 따로 띄우려면 `pnpm dev:web` /
`pnpm dev:server`, 워크스페이스 점검은 `pnpm typecheck` / `pnpm build`. 설계
문서는 [`docs/design`](./docs/design/README.md)에 있습니다.

## 기여

PR 환영합니다 — 버그 수정, 밸런스 조정, 새 무기·클래스, 또는 몇 판 해보고 남기는
피드백까지.

1. `dev`에서 브랜치를 만듭니다 (예: `feat/your-thing`).
2. 게임플레이는 **서버 권위**를 유지하세요. 클라이언트는 의도를 보내고 결과는
   서버가 결정합니다. 새 클래스·무기는 `@vertix/shared`에 정의합니다.
3. `pnpm typecheck`와 `pnpm build`를 돌린 뒤 `dev`로 PR을 엽니다.
